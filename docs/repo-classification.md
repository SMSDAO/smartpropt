# Repository Classification — SmartPromts

## Classification: MIXED_PLATFORM ⚠️ VIOLATION

The SmartPromts repository currently contains **two distinct platform targets** in a single repo. This is a structural violation requiring platform separation.

## Violation Details

### VIOLATION: Mixed Platform Deployment Risk

**Problem:** The `admin-desktop/` directory contains Tauri (Rust + desktop binary) code co-located with the web application. This causes:

1. **Vercel deployment risk** — Tauri code must never be deployed to Vercel
2. **Dependency confusion** — Desktop dependencies mixed with web dependencies
3. **CI complexity** — Build pipelines must explicitly exclude desktop code
4. **Security surface** — Admin desktop shares the same repo as public web code

**Mitigation applied:**
- `vercel.json` does not reference `admin-desktop/`
- `tsconfig.json` excludes `admin-desktop/`
- Vercel deployment only builds `/app` and Next.js routes

**Required action:**
- Move `admin-desktop/` to new repository: `SMSDAO/SmartPromts-Admin`
- Remove `admin-desktop/` from this repository after migration
- See `/docs/admin-desktop.md` for integration documentation
